1

⬇️ Baixar os arquivos

A skill storm-research é composta por dois arquivos de texto: o SKILL.md (o pipeline de 4 fases) e o report-template.html (o molde do relatório). Baixe os dois — não há instalador, dependência ou serviço pago. Escolha o idioma: PT-BR ou EN entregam o mesmo método, só muda a língua dos prompts internos.

Nenhum desses passos é "instalar" no sentido tradicional: você baixa dois arquivos de texto (passo 1), coloca-os na pasta que o agente varre (passo 2) e chama a skill em linguagem natural (passo 3). Os tópicos a seguir detalham cada passo.

⬇️ Baixe a skill e o template

Os dois arquivos que compõem a skill storm-research. No próximo tópico você os solta em .claude/skills/storm-research/ e a skill funciona — sem dependências, scripts ou serviços pagos.

Escolha PT-BR ou EN. Não precisa baixar os dois idiomas — um par (SKILL.md + template) já basta.

O prompt-mestre do pipeline STORM em 4 fases.

💡 Dica — PT ou EN?

Escolha o idioma em que você lê com conforto: ele afeta a língua dos prompts das cinco lentes e dos títulos do relatório. O método, as fases e a verificação são idênticos. Se for compartilhar o briefing com um público em inglês, baixe a versão EN.

C

2

📁 Onde colocar

Os dois arquivos moram juntos numa pasta com o nome da skill, dentro de ~/.claude/skills/. O Claude Code varre essa pasta na inicialização e descobre a skill pela description do frontmatter. Um detalhe importante: dentro da pasta, o arquivo de instruções deve se chamar SKILL.md (sem o sufixo .pt/.en) e o molde, report-template.html. Os sufixos de idioma existem só para você diferenciar no download.

🎯 Objetivo: criar a pasta da skill e mover os dois arquivos baixados para dentro dela, já com os nomes finais (SKILL.md e report-template.html).

terminal · bashcopie e rode
mkdir -p ~/.claude/skills/storm-research

mv <PASTA_DOWNLOADS>/storm-research.SKILL.<pt>.md \
   ~/.claude/skills/storm-research/SKILL.md

mv <PASTA_DOWNLOADS>/report-template.<pt>.html \
   ~/.claude/skills/storm-research/report-template.html

Troque as partes em amarelo: <PASTA_DOWNLOADS> pela pasta onde o navegador salvou (ex.: ~/Downloads) e <pt> por pt ou en, conforme o idioma que você baixou. O mv já renomeia para os nomes finais.

✅ Como verificar: liste o conteúdo da pasta.

ls ~/.claude/skills/storm-research
# esperado:  SKILL.md   report-template.html

Se os dois nomes aparecem, a skill está no lugar.

✓ Pasta correta

  • ✓Os dois arquivos JUNTOS na mesma pasta
  • ✓Pasta com o nome storm-research
  • ✓Arquivo de instruções chamado SKILL.md

✗ Onde costuma dar errado

  • ✗Arquivos espalhados em pastas diferentes
  • ✗Manter o nome ...SKILL.pt.md
  • ✗Esquecer o report-template.html
3

🔀 Outros agentes

Como a skill é só texto + ferramentas nativas, ela é portátil: roda em outros agentes de linha de comando que seguem a mesma convenção de skills. Você não muda nada nos arquivos — só a pasta-raiz muda. A pasta interna skills/storm-research/ e os dois arquivos continuam idênticos.

Claude Code

~/.claude/skills/storm-research/

Codex

~/.codex/skills/storm-research/

Outros agentes

~/.agents/skills/storm-research/

🧳 Por que isso funciona

A storm-research não chama nenhum script, API ou serviço pago. Ela usa apenas o que qualquer agente compatível já oferece: disparar subagentes, buscar/abrir páginas na web e gravar arquivos. Por isso o mesmo par de arquivos é "instalável" em qualquer um deles — basta colocá-los na pasta de skills do agente que você usa.

4

✅ Verificar a instalação

Antes de disparar uma execução de verdade, confirme que o agente "enxerga" a skill. A forma mais simples é abrir o Claude Code na pasta onde você está trabalhando e pedir em linguagem natural que ele liste as skills disponíveis.

🎯 Objetivo: confirmar que o Claude Code reconhece a storm-research e leu seu frontmatter.

cole no Claude Codeprompt
Liste as skills disponíveis e confirme que "storm-research"
aparece. Mostre a descrição dela.

✅ Como verificar: o Claude responde listando as skills e cita storm-research — em geral repetindo a description do frontmatter (algo como "rodar Storm Research / briefing em HTML com múltiplas perspectivas"). Se a skill não aparece, revise a pasta e os nomes dos arquivos (tópico 6) e reinicie a sessão.

💡 Dica

Se você acabou de mover os arquivos com a sessão já aberta, o agente pode não ter relido a pasta de skills. Feche e reabra o Claude Code para ele varrer .claude/skills/ de novo.

5

🎯 Dar contexto

O STORM mira a seção prática — o "insight acionável" — no leitor-alvo que ele identifica na Fase 0. Se você não disser nada, ele assume "um profissional ou tomador de decisão na área". Mas, se você contar quem é e qual seu negócio, todo relatório sai direcionado ao seu papel, em vez de a uma pessoa genérica.

exemplo de linha de contextocole antes de rodar
Sou <fundador de uma agência de automação em n8n>.
Quando rodar storm-research, mire o insight acionável
no meu papel: <o que eu preciso decidir>.

Troque as partes em amarelo pelo seu contexto real. Quanto mais específico o papel, mais útil fica a seção de movimentos acionáveis do relatório.

💡 Dica — contexto persistente

Se você roda STORM com frequência, vale registrar esse contexto no seu CLAUDE.md do projeto. Assim todo relatório já nasce mirado no seu papel, sem precisar repetir a linha de contexto a cada execução. (Personalização mais profunda fica para a Trilha 3.)
